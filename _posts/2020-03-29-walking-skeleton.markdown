I like the idea of starting the project by implementing a tiny feature and test it end to end.
I mean really end to end - including building the app, spinning up all necessary docker containers,
running the acceptance test of the feature, closing and cleanning.
It requires a decent amount of thought and work put into it to make the test run automatically.
But in my opinion it pays off, because it makes me thouroughly pick all the pieces of the application stack,
and put it all together.
It's of course very far from being incremental design, but I prefer to know what technologies I'm going to
work in. I think it saves a lot of work in the first deployment stage, when it may appear that some
technologies that I've thought would work together, are completely incompatible.
